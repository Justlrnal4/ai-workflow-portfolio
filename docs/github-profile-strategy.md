# GitHub Profile Strategy

This GitHub profile is intended to be a public proof-of-work page, not a full inventory of every project or private repo.

## Visible structure

1. Profile README — explains who Justin is and where to start.
2. ai-workflow-portfolio — main public hub for selected work and case studies.
3. BurgReport — one public build sample.
4. aX Gateway PR/fork — optional public contribution proof.
5. Private contribution graph — activity signal without exposing private repo names.

## What belongs on the public profile

- Clean project summaries
- Safe public links
- Selected proof-of-work
- Public PRs
- Public build samples
- High-level technical stack

## What does not belong on the public profile

- Private code
- Secrets or environment files
- Client-confidential work
- Local machine paths
- Messy unfinished repos
- Overstated claims
- Claims that private projects are public production software
